weibo爬虫学习

版本记录时间为2023年10月九日可用，学习的项目源自github上的weibo_crawl爬虫项目。

学习到的知识：

  1. codecs库：

    1. codecs库专门用作编码转换，当我们要做编码转换的时候可以借助codecs很简单的进行编码转换。

    2. codecs模块提供一个open方法，三个参数encoding, errors, buffering，这三个参数都是可选参数，但是对于应用来说，需要明确指定encoding的值，而errors和buffering使用默认值即可。使用方法如下：

    3. 常用函数：

      1. codecs.open()
        codecs.open(filename, mode='r', encoding=None, errors='strict', buffering=1)

        使用给定的 mode 打开已编码的文件并返回一个 StreamReaderWriter的实例，提供透明的编码/解码；与内置函数open类似。

      2. codecs.encode()

        codecs.encode(obj, encoding='utf-8', errors='strict')
        使用为 encoding 注册的编解码器对 obj 进行编码，类似str.encode()方法。

      3. codecs.decode()

        codecs.decode(obj, encoding='utf-8', errors='strict')
        使用为 encoding 注册的编解码器对 obj 进行解码，类似str.decode()方法。

      4. codecs.lookup()

        codecs.lookup(encoding)

        在 Python 编解码器注册表中查找编解码器信息，并返回一个 CodecInfo 对象。

      5. codecs.getencoder()

        codecs.getencoder(encoding)

        查找给定编码的编解码器并返回其编码器函数。

    4. codecs — 编解码器注册和相关基类 — Python 3.12.0 文档

  2. csv库

    ​ CSV是一种以逗号分隔数值的文件类型，在数据库或电子表格中，常见的导入导出文件格式就是CSV格式，CSV格式存储数据通常以纯文本的方式存数数据表。

    1. csv 库是专门处理csv文件的库，与数据库的操作方法有些类似。其中有4个常用的对象：

      • csv.reader：以列表的形式返回读取的数据。
      • csv.writer：以列表的形式写入数据。
      • csv.DictReader：以字典的形式返回读取的数据。
      • csv.DictWriter：以字典的形式写入数据。

    2. 一张操作表格：

      姓名 年龄 职业 家庭地址 公资
      张三
      李四
      王五
      刘六

      1. 数据读取储存依靠row获取行以及列数据，比如

        姓名 年龄 职业 家庭地址 公资
        张三
        李四
        王五
        刘六
        曹操 30 皇帝 中国 0

      2. 文件写入数据的时候注意要aim_file=open('name.csv','w',newline=``'')

        不能缺少newline，不然会形成中间的空行。

      Pandas CSV 文件 | 菜鸟教程 (runoob.com)

    4. 官方文档：csv — CSV 文件读写 — Python 3.12.0 文档

具体流程：

  1. 创建日志文件以便记录日志信息，加载config配置项文件初始化Class weibo来及进行最基本的配置

  2. 运行main主函数。跳转到def get_config()，获取config.json文件信息。

  3. 进行wb = Weibo(config)类的初始化，然后进入Weibo的类中的函数start开始wb.start()爬取信息。

  4. 判断爬取的列表数量，分析数量之后开始def initialize_info(self, user_config)，初始化爬虫信息。初始化信息后，进行def get_pages(self)来获取全部微博。

  5. 获取时首先进行const等配置并进行日期的计算记录，根据起始日期执行def get_page_count(self)来获取微博页数，获取后进行配置，循环执行def get_one_page(self, page)来获取一页全部的微博。

  6. 其中首先执行def get_weibo_json(self, page)来获取网页中微博json数据，循环过程当中之后对于其中的一条微博执行def get_one_weibo(self, info)来获取一条微博的全部信息。并在每爬取20页执行write_data(wrote_count)写入文件。

    1. write_data(wrote_count)写入文件

  7. 一条微博的分类为转发或者原创，若为转发，若为长微博，则执行get_long_weibo(weibo_id)，若为短微博，则执行parse_weibo(weibo_info)。

  8. 最终都进入到parse_weibo(weibo_info)来进行微博的解析。将微博设置为有序字典，对于单条微博进行配置，并获得微博的信息写入json。

  9. 返回self.standardize_info(weibo)来标准化信息，去除乱码。

  10. 对于单条微博的解析全部完成之后，返回到def get_one_page(self, page)进行cookie的检验，检验通过之后对于置顶微博进行判断等。整体流程基本结束。
